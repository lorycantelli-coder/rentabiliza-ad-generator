# Story 1.1 — Backend Stability

**Status:** Draft
**Created:** 2026-03-07
**Epic:** Rentabiliza Ad Generator MVP

---

## Description

Garantir que o backend dos 3 copywriters (Halbert, Wiebe, Bencivenga) seja robusto e production-ready. Atualmente o backend (`backend.js`) é funcional mas falta error handling, logging estruturado e observabilidade.

**Current State:**
- ✅ 3 agents rodando em paralelo via Claude API
- ❌ Sem tratamento de timeouts/rate limits
- ❌ Sem retry logic
- ❌ Sem logging estruturado
- ❌ API key hardcoded no arquivo
- ❌ Sem tests

---

## Acceptance Criteria

- [x] **AC1:** API key lida do `.env.local` (não hardcoded)
- [x] **AC2:** Retry logic implementado (3 tentativas com exponential backoff)
- [x] **AC3:** Timeout tratado (fallback gracioso - 30s timeout)
- [x] **AC4:** Logging estruturado (arquivo + console, níveis: ERROR, WARN, INFO, DEBUG)
- [x] **AC5:** Health check endpoint (`GET /health` → status dos agents)
- [x] **AC6:** Unit tests para cada agent (100% pass)
- [x] **AC7:** Error responses estruturadas (com error codes)

---

## Scope

**IN:**
- Remover hardcoded API key
- Adicionar error handling robusto
- Implementar retry com backoff
- Adicionar logging (pino ou winston)
- Health check endpoint
- Unit tests básicos

**OUT:**
- Analytics/dashboards (Story 1.3)
- Frontend changes (Story 1.2)
- Deployment/Docker (Story 1.4)

---

## Complexity

**Estimate:** 5 story points
**Dimensions:**
- Scope: 3/5 (3 agents + logging)
- Integration: 2/5 (Claude API only)

- Knowledge: 2/5 (standard patterns)
- Risk: 3/5 (API changes would break agents)
